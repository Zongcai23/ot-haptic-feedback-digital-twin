# Repository Customisation

This repository is meant to be easy to adapt before public release.

## What to customise before pushing to GitHub
- Replace placeholder text in the README if needed.
- Add your real pose/depth notebook under `code/vision/`.
- Add checkpoints only if they are legally and practically redistributable.
- Add a short demo GIF only if the file size is small enough for GitHub.
- Keep large videos external if necessary.

## Recommended pattern for paper-code repositories
A clean conference repository usually keeps:
1. a concise README with one figure and one paragraph;
2. a code map that tells readers where the key files are;
3. a minimal reproducible tutorial with exact commands;
4. a paper PDF and citation block;
5. optional assets such as figures or small GIFs.

That is the pattern followed here.
